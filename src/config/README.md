# 游戏配置文件说明

本目录包含游戏的所有配置常量，按功能模块组织。

## 📁 配置文件列表

### 🎨 `colors.js` - 颜色配置
定义游戏的赛博朋克/霓虹色彩方案：
- 友方单位配色（青色系）
- 敌方单位配色（洋红/紫色系）
- 武器塔配色（火箭塔-紫色，激光塔-绿色）
- UI和系统配色

### 📐 `layout.js` - 布局配置
定义游戏空间布局相关参数：
- 画布基准尺寸（1600x640）
- 网格系统（CELL_SIZE = 80px）
- 区域划分（顶部UI、战斗区、武器容器）
- 响应式布局计算函数

### 🎮 `gameplay.js` - 游戏玩法配置
定义游戏平衡性和核心玩法数值：
- **基础武器属性**：射速、攻击范围、子弹属性
- **火箭塔配置**：成本、伤害、追踪参数、升级数据
- **激光塔配置**：成本、射速、激光束效果、升级数据
- **经济系统**：初始金币、成本、收益、奖励
- **武器通用配置**：等级上限、动画参数、血量条

### 👾 `enemies.js` - 敌人配置
定义敌方单位的属性和行为：
- **基础敌人属性**：尺寸、速度、生命值、攻击力
- **普通坦克配置**：移动、攻击、子弹属性
- **声波坦克配置**：增强属性、声波攻击参数
- **敌人管理器配置**：波次系统、生成间隔、难度递增
- **动画配置**：待机动画、受击效果

### 🔫 `weaponTypes.js` - 武器类型定义
定义可建造的武器类型元数据：
- 武器ID和名称
- 图标和描述
- 成本和收益
- 颜色主题
- 工具类（WeaponConfig）：提供查询和计算功能

### 🎨 `visual.js` - 视觉效果配置
定义游戏中的视觉元素参数：
- **阴影效果**：多层阴影透明度
- **选中光环**：多层光环尺寸、线宽、透明度
- **粒子效果**：爆炸和枪口闪光粒子数量
- **激光束效果**：多层激光束宽度和透明度
- **武器塔尺寸比例**：各部件相对尺寸
- **敌人坦克尺寸比例**：各部件相对尺寸

### 🖥️ `ui.js` - UI配置
定义用户界面视觉参数：
- **金币显示**：字体、内边距
- **小地图**：尺寸、圆角、边框、标记点
- **波次显示**：字体、偏移
- **操作按钮**：尺寸、圆角、字体
- **波次通知**：动画、面板、光晕效果
- **帮助界面**：标题、正文、按钮参数
- **开始界面**：遮罩、标题、按钮配置

## 📋 使用方式

### 方法 1：从 constants.js 统一导入（推荐）

```javascript
import { 
  COLORS, 
  CELL_SIZE, 
  LASER_DAMAGE,
  ENEMY_MOVE_SPEED 
} from '../constants';
```

### 方法 2：从具体配置文件导入

```javascript
import { COLORS } from '../config/colors';
import { LASER_DAMAGE } from '../config/gameplay';
```

## 🎯 设计原则

1. **集中管理**：所有魔法数字提取为常量
2. **分类组织**：按功能模块划分配置文件
3. **清晰命名**：使用全大写+下划线的命名约定
4. **详细注释**：每个常量都有中文注释说明
5. **易于维护**：修改配置时无需改动业务代码

## 🔧 配置调整指南

### 调整游戏难度
- `gameplay.js`: 修改武器伤害、成本
- `enemies.js`: 修改敌人血量、速度、生成间隔

### 调整视觉效果
- `colors.js`: 修改配色方案
- `visual.js`: 修改光效、阴影、尺寸比例
- `ui.js`: 修改UI元素尺寸和样式

### 调整游戏平衡
- `gameplay.js`: 调整经济系统、武器升级数据
- `enemies.js`: 调整波次难度递增率

### 调整布局
- `layout.js`: 修改画布尺寸、网格大小、区域划分

## 📊 常量命名规范

- **尺寸/位置**：`XXX_WIDTH`, `XXX_HEIGHT`, `XXX_OFFSET_Y`
- **颜色**：`XXX_COLOR`, 使用十六进制值 `0xRRGGBB`
- **时间/速度**：`XXX_DURATION`, `XXX_INTERVAL`, `XXX_SPEED`
- **比例/倍数**：`XXX_RATIO`, `XXX_MULTIPLIER`, `XXX_ALPHA`
- **数量/数值**：`XXX_COUNT`, `XXX_DAMAGE`, `XXX_HP`
- **阈值**：`XXX_MAX`, `XXX_MIN`, `XXX_THRESHOLD`

## 🔄 配置更新流程

1. 在对应配置文件中添加/修改常量
2. 确保在 `constants.js` 中导出（如果新增配置文件）
3. 在业务代码中使用新常量
4. 测试验证效果
5. 更新本 README 文档

---

💡 **提示**：修改配置后，建议使用 `npm run dev` 热重载查看效果！
